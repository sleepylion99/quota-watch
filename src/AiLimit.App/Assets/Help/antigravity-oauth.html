Antigravity OAuth setup guide

Create an OAuth client so Quota Watch can read Antigravity usage from Google Cloud.

Optional: You do not need this setup when Antigravity IDE is running.

Setup overview

  1. Select or create a project→
  2. Register Google Auth→
  3. Create client→
  4. Choose Desktop app→
  5. Copy ID and Secret→
  6. Save in Quota Watch

Step-by-step instructions

  1. Select or create a Google Cloud project

    Sign in to Google Cloud Console and use the project selector at the top. Choose an existing project or create a new one.

    Search terms: Project selector, New project

  2. Complete Google Auth registration if needed

    For a project using OAuth for the first time, follow Google Auth Platform to register the app name and support email. Skip this step if registration is already complete.

    Menu or search terms: Google Auth Platform, Branding, Audience
    Important: If the OAuth consent screen stays in Testing mode, refresh tokens expire after 7 days and Quota Watch will fail to refresh. Add your own Google account under Audience > Test users, or set Publishing status to In production.

  3. Create an OAuth client

    Open Clients and select Create client.

    Path: Google Auth Platform → Clients → Create client

  4. Choose the application type

    For Application type, choose Desktop app, enter a recognizable name, and create the client.

    Required selection: Application type: Desktop app

  5. Copy Client ID and Client Secret

    Copy the Client ID and Client Secret shown after creation and keep them in a secure place.

    The full Client Secret may only be visible when it is created. Store it securely before closing the window.

  6. Paste the values into Quota Watch and save

    In Quota Watch settings, find Antigravity OAuth, paste the Client ID and Client Secret, and save. Refresh to check the status.

    Quota Watch: Settings → Antigravity OAuth → Save
    Your Client ID and Secret are encrypted with Windows DPAPI and stored only on this device. They are never sent anywhere except Google's token endpoint when refreshing.

Troubleshooting

Symptom Likely cause Fix
Refresh fails after about 7 days Consent screen is still in Testing mode Add your account under Test users, or set Publishing status to In production.
invalid_client / HTTP 401 Client ID or Secret is wrong, or the Secret was rotated Re-copy both values from the same OAuth client and save again.
access_denied Your Google account is not allowed for this client Add the account as a Test user, or sign in with the account that owns the project.
Open Google Cloud Clients Open Google official help
